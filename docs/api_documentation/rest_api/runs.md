# Runs API

REST API endpoints for managing experiment runs.

## Endpoints

### List Runs

`GET /api/runs` - List runs with filtering, sorting, and pagination

### Get Run

`GET /api/runs/{run_id}` - Get run details

### Create Run

`POST /api/runs` - Create a new run

### Update Run State

`PATCH /api/runs/{run_id}/state` - Update run state

### Delete Run

`DELETE /api/runs/{run_id}` - Delete a run

See the [REST API Guide](../../getting_started/rest_api.md) for complete examples.
